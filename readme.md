## Описание

Проект состоит из 2-ух частей: бекенда и фронтенда

- Бекенд: ASP.NET Core Web API, с подключением к базе данных (MSSQL) через EF Core.
- Фронтенд: ASP.NET Core MVC

### Бекенд

Состоит из 5 частей

- Domain - сущности и интерфейсы для репозиториев
- Infastructure - реализация доступа к данным через EF Core
- Application - бизнес-логика
- API - контроллеры и точки входа
- Tests - модульные тесты для бизнес-логики (конкретно, для расчёта встреч). Тестовый кейс из примера там тоже есть.

#### Особенности:

- Использует архитектурный паттерн DDD - Domain Driven Design; Думаю, разобраться будет не сложно ;)

- Сущности и DTO храняться в CoExittor.Common - их могут одновременно использовать и фронтенд, и бекенд.

- Авторизация и регистрация - в контроллере UserController. Cookies издаются в бекенде, и передаются на фронтенд.
Можно было бы сделать лучше, но вот такой вышел у меня SSO для нищих.

### Фронтенд

Обычный ASP.NET Core MVC, с контроллерами и формами. Ничего особенного.
С версткой часто помогал ChatGPT, и bootstrap с js тоже он подключал.

#### Особенности:

- Авторизация и регистрация: позволяет смотреть списки записей, и указывает при создании события, кто его создал.
Изменений и удаления событий в зависимости от пользователя нет, да и не надо?
